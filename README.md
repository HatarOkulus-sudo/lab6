# Лабораторная работа 5 — консольный менеджер коллекции StudyGroup

Вариант: **2524**

Консольное Java-приложение для управления коллекцией объектов `StudyGroup`.
Проект использует архитектуру на основе команд и поддерживает загрузку/сохранение данных в XML.

## Возможности

- Управление коллекцией в памяти (`LinkedList<StudyGroup>`)
- Диспетчер команд (`help`, `info`, `show`, `add`, `remove_by_id`, `clear`, `save`, `exit`)
- Интерактивный ввод сложных объектов (`StudyGroup`, `Person`, enum-типов)
- Сериализация/десериализация XML без сторонних библиотек

## Структура проекта

- `src/app` — класс запуска приложения (`ConsoleApp`)
- `src/commands` — реализации команд
- `src/data` — модели предметной области и enum-ы
- `src/managers` — менеджеры коллекции, файла, ввода и команд

## Требования

- JDK 17+ (или совместимая версия)
- IntelliJ IDEA (рекомендуется)

## Сборка

Скомпилировать все исходники из корня проекта:

```bash
javac -d out $(find src -name "*.java")
```

## Запуск

В `ConsoleApp` нет метода `main`, поэтому запуск выполняется через класс-точку входа (где объявлен `public static void main(String[] args)`).

Если нужно, создайте класс запуска, который инициализирует:

- `CollectionManager`
- `FileManager`
- `InputManager`
- `CommandsManager` через `CommandBuilder`
- `ConsoleApp`, затем вызовите `run()`

## Файл данных

Путь к XML-файлу задается при создании `FileManager`.

Типичный сценарий:

1. Инициализация менеджеров
2. Загрузка коллекции из XML
3. Запуск консольного цикла
4. Выполнение команд
5. Сохранение через `save`

## Примечания

- `exit` завершает программу без автоматического сохранения.
- Перед `exit` используйте `save`, чтобы сохранить данные.
